# Minimum Viable Product (MVP)

## Overview
A product development philosophy focused on testing core value propositions with the minimum functional set required for hypothesis validation. It is a model that prioritizes **「speed of learning」** over initial **「completeness」**.

## Evaluation (1–5)
**Versatility**:: 4  
**Immediacy**:: 4  
**Difficulty**:: 2  
**Misuse Risk**:: 4  

### Editorial Comments
**Comment**:: An extremely powerful tool, but misunderstood as simply **「releasing unfinished work」**, it can lead to brand damage.

## The First Question
- "What is the bare minimum we can build to sufficiently validate this hypothesis at the lowest cost?"

### Objectives
- Maximize the speed of learning.
- Prevent unnecessary over-engineering.

### Pitfall Questions
- "Let's wait until it's perfect before we release it."
- "It won't be valued unless we include every feature."
- "Let's try it only after we are certain it won't fail."

## How to Use (Mini-Steps)
1. **Clarify** the hypothesis you want to validate.  
2. **Narrow down** to a single essential feature.  
3. **Launch** with the minimum configuration.  
4. **Capture** data and learn from the results.

## Use Cases
- **Business**: Launching new services.  
- **Daily Life**: Testing new habits or career experiments.  
- **Decision Making**: Small-scale validation before making large investments.  

## Common Misuses
- Neglecting quality entirely.  
- Releasing without a structured learning design.  
- Failing to define KPIs.
