# Mental Model Atlas への貢献ガイド

「日本語圏初の体系的なメンタルモデル辞書」を共に創り上げてくださる皆さんの力を必要としています。
本プロジェクトは、単なる用語集ではなく、意思決定を支える「知的OS」を目指しています。

## 🤝 貢献の種類
1. **モデルの新規追加**: 未掲載のメンタルモデルの執筆。
2. **コンテンツのブラッシュアップ**: 既存モデルの誤用例、実践例の追記。
3. **英語翻訳の改善**: 自然な英語表現への修正。
4. **技術的貢献**: UI/UXの改善、パフォーマンス最適化。

## 📝 コンテンツ執筆定義表
モデルの品質と一貫性を保つため、以下の定義に基づいて Front Matter を記述してください。

### 1. Layer（階層）の選択基準
モデルが「どの抽象レベル」に位置するかを定義します。

| Layer | 定義 | 代表的なモデル例 |
| :--- | :--- | :--- |
| **META** | 思考のOS。認識の枠組みそのものを変えるもの。 | 第一原理思考、システム思考 |
| **CORE** | 汎用的なフレームワーク。多様な場面で応用可能。 | MECE、トレードオフ、反証可能性 |
| **TOOL** | 特定のタスクに特化した具体的な道具。 | SWOT分析、ファイブフォース、MVP |

### 2. Viewpoint（活用場面）のタグ
どの文脈で有用かを定義します（複数選択可）。

| タグ | 名称 | 活用シーンのイメージ |
| :--- | :--- | :--- |
| **DM** | 意思決定 | 二者択一、優先順位付け、投資判断 |
| **PS** | 問題解決 | 原因究明、ボトルネック特定、構造化 |
| **NB** | 新規事業 | 市場分析、仮説検証、価値提案 |
| **LT** | 論理思考 | 議論の構築、説得、情報の整理 |
| **LIFE** | 人生・キャリア | 習慣形成、対人関係、自己理解 |

### 3. Score（評価スコア）の判定基準
各項目 **1（低）〜 5（高）** で評価します。

| 項目 | 1 の目安 | 5 の目安 |
| :--- | :--- | :--- |
| **応用度** | 特定の専門職のみで有効 | あらゆる日常・仕事で有効 |
| **即効性** | 成果が出るまで数年かかる | 知った直後の判断から使える |
| **理解難易度** | 直感的に理解できる | 数学・哲学的な深い学習が必要 |
| **誤用リスク** | 悪影響がほとんどない | 使い方を誤ると致命的な判断ミスを招く |

## 🛠 執筆のルール
- **Markdown形式**: ボールド（太字）と鍵括弧「」を組み合わせる際は、**強弱記号（`**`）と鍵括弧の間にスペース** を入れてください（例： `** 「〇〇」 **`）。
- **誤用例（Antipatterns）**: 読者が「わかったつもり」で失敗しないよう、実際の現場で起きがちな誤用を最低1つは記載してください。
- **日英併記**: 本プロジェクトは日英2言語展開です。日本語版を作成した際は、可能な範囲で英語版（`en/` フォルダ）の作成も検討してください。

## 🚀 ワークフロー
1. **Issueを確認**: 重複を避けるため、既存のIssueを確認してください。
2. **Fork & Branch**: `feature/add-[model-name]` ブランチで作業。
3. **Pull Request**: テンプレートに従ってPRを作成。
